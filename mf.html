SPECIFICATION

MFクラウド勤怠 RPA連携
自動勤怠チェックシステム 仕様書

設計の基本方針

MFクラウド勤怠からエクスポートされたCSVを基に、最新の勤怠状況をスプレッドシートへ反映します。常に最新の一覧性を保ちつつ、新規に検知された不備のみをSlackへ通知する、効率的な管理体制を構築します。

Primary Mode
RPA + GAS Hybrid
Sync Strategy
Full Data Refresh
RPA OPERATIONS

RPAデータ連携フロー

実行ステップ (深夜自動実行)

  • 1. **ログイン**: 管理者アカウントでMFクラウド勤怠にログイン。
  • 2. **画面遷移**: [全権管理者メニュー] > [連携] > [エクスポート] > [出勤簿] を選択。
  • 3. **条件設定**: 対象期間（前月分または当月分）を指定し、CSV出力をリクエスト。
  • 4. **ダウンロード**: 生成されたCSVファイルを一時保持。
  • 5. **Drive配置**: 指定したGoogle Driveフォルダへアップロード（既存ファイルは上書き/置換）。

最新のCSVデータを読み込み、以下の項目をシートへマッピングします。

LOGIC

取得対象期間の定義

CSV出力時の期間指定

毎月 1日 ～ 5日

対象: 前月フル期間

目的: 締め処理に向けた前月データの最終確認。

毎月 6日 ～ 末日

対象: 当月1日 ～ 前日

目的: 月中に発生した不備の早期発見・是正。

DATA STRUCTURE

シート構成

1. 日次勤怠シート

※同期のたびに clearContents() を行い、全件を再出力します。

列カラム名役割
A-ECSV抽出データ最新の打刻・休暇状況をそのまま反映
F判定結果最新の状態に基づき「正常」「打刻漏れ」を再判定

2. 勤怠不備シート

不備が発生しているレコードのみを表示する**「現時点の不備リスト」**です。

  • データ同期: 毎回クリアされ、最新のCSVで見つかった不備のみが表示されます。
  • 通知制御: スプレッドシート上は更新されますが、Slack通知時には「通知履歴」を別途参照し、同じ不備を二度通知しないよう管理します。
ANALYSIS

不備判定ロジック

打刻実績勤怠状況(休暇等)判定結果（シート表示用）
あり (両方)なし / 半休正常勤務
なし (両方)全日休暇あり有給休暇等
片方のみ空なし / 半休出勤打刻漏れ / 退勤時打刻漏れ
なし (両方)なし / 半休出退勤打刻漏れ
あり (打刻あり)全日休暇あり有給申請日に打刻あり

セキュリティ設計

RPA Credentials

Data Source
