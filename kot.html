KING OF TIME API 連携
自動勤怠チェックシステム 仕様書

設計の基本方針

KING OF TIME APIから取得した「打刻実績」と「休暇申請」を突合し、勤怠不備を自動検知します。反映済みの休暇だけでなく、承認待ちの申請も別途取得することで、誤検知のない精緻なチェック体制を実現します。

Primary Mode
Daily Auto-Trigger
Logic Principle
Multi-Endpoint Cross Check
SPECIFICATION 01

日別勤務データ取得仕様

Endpoint
GET /v1.0/daily-workings

※本APIで取得できる `leaveName`（休暇名）は承認・反映済みのものに限定されるため、実績確定用として利用します。

項目名JSONキー役割
勤務日dateyyyy-MM-dd。突合の主キー。
従業員キーemployeeKey従業員特定のID。
打刻実績workingTimes[]出勤(start)・退勤(end)時刻の有無を判定。
反映済休暇leaveName反映済みの休暇名。
SPECIFICATION 02

休暇申請取得仕様

Endpoint
GET /v1.0/leave-requests

【必須】承認待ちの申請データを取得します。実績データのみでは、未承認の休暇を「無断欠勤」と誤判定する可能性があるため、本APIでの補完が不可欠です。

項目名JSONキー役割
申請対象日date実績データの date と突合。
休暇名leaveName有給休暇等の具体的な名称。
承認状況status1 (承認待ち) のデータを「申請中」として受理。
SPECIFICATION 03

従業員データ取得仕様

Endpoint
GET /v1.0/employees

勤務データには姓名が含まれないため、実行開始時に本APIを一度呼び出し、employeeKeyと姓名のマッピング（Map）を作成します。

項目名キー役割
従業員キーkey各データの employeeKey と突合。
姓名lastName / firstName表示用姓名の構成。
LOGIC

取得対象期間の定義

実行日による自動切り替え

毎月 1日 ～ 5日 (前月是正フェーズ)

対象: 前月1日 ～ 前月末日

目的: 締めチェック。 確定に向けたデータの最終整合性確保。

毎月 6日 ～ 末日 (当月是正フェーズ)

対象: 当月1日 ～ 実行日前日

目的: 日次是正。 発生した不備を即時通知。
