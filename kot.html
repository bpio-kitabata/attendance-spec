KING OF TIME API 連携
自動勤怠チェックシステム 仕様書

設計の基本方針

KING OF TIME APIから取得した「日別勤務データ」を基に、打刻漏れや休暇不備を自動検知します。本エンドポイント単体で、打刻実績と休暇情報（全休・半休・時間休）を統合的に取得し、全自動で是正フローを回します。

Primary Mode
Daily Auto-Trigger
Sync Logic
Consolidated Endpoint Sync
SPECIFICATION 01

日別勤務データ取得仕様

Endpoint (カテゴリ: 日別データ)
GET /v1.0/daily-workings

打刻の有無に加え、全日休暇、半日休暇、および時間休暇（hourHolidays）の情報を一括で取得します。

項目名JSONキー役割
勤務日dateyyyy-MM-dd。突合の主軸。
従業員キーemployeeKey従業員特定のID。
打刻実績workingTimes[]出勤(start)・退勤(end)時刻の有無。
全休・半休leaveName反映済みの休暇区分（有給休暇、有休(午前)など）。
時間休暇hourHolidays[]取得した時間休の名称（name）、開始、終了時刻。
SPECIFICATION 02

従業員データ取得仕様

Endpoint (カテゴリ: 従業員)
GET /v1.0/employees

勤務実績データには employeeKey しか含まれないため、本APIを用いて表示用の姓名（lastName / firstName）をマッピングします。

項目名キー役割
従業員キーkey各データの employeeKey と突合。
姓名lastName / firstName表示用姓名。
LOGIC

取得対象期間の定義

実行日による自動切り替え

毎月 1日 ～ 5日 (前月是正フェーズ)

対象: 前月1日 ～ 前月末日

目的: 締めチェック。データの最終整合性確保。

毎月 6日 ～ 末日 (当月是正フェーズ)

対象: 当月1日 ～ 実行日前日

目的: 日次是正。発生した不備を即時通知。

ANALYSIS

不備判定ロジック

打刻の有無と、休暇情報（全休・半休・時間休）を突合して判定を下します。

打刻実績休暇情報(全休・半休・時間休)判定結果（通知用）
あり (両方)なし / 半休 / 時間休正常勤務
なし (両方)全日休暇あり有給休暇等
片方のみ空なし / 半休 / 時間休出勤打刻漏れ / 退勤時打刻漏れ
なし (両方)なし / 半休 / 時間休出退勤打刻漏れ
あり (打刻あり)全日休暇あり有給申請日に打刻あり

※半休や時間休の場合、残りの勤務時間に打刻が必要なため、打刻がない場合は「打刻漏れ」として検知します。
