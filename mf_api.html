SPECIFICATION V2.0

MFクラウド勤怠 API連携
自動チェックシステム 概要

システムの目的

本システムは、MFクラウド勤怠 APIキー認証を用い、全従業員の勤務状況を自動で監視します。RPAによるCSV手動ダウンロードを完全に廃止し、APIベースの疎結合なデータパイプラインにより、打刻不備のリアルタイム検知を実現します。

取得したデータはGoogle Apps Script(GAS)で解析され、条件に合致する不備のみがSlackへ通知されます。

TECHNICAL DETAILS

02. 勤務データ取得
daily-workings API

利用エンドポイント

GET /daily_attendances

※「daily-workings」はシステム内部での処理単位としての呼称です。

主要な取得フィールド

フィールド判定ロジック
clock_in_at出勤打刻。nullの場合は「出勤打刻漏れ」
clock_out_at退勤打刻。nullの場合は「退勤打刻漏れ」
attendance_type_name「平日」「有給休暇」等のステータス判別
work_pattern_nameパターンの有無で「本来の勤務日」を特定
TECHNICAL DETAILS

03. 従業員データ取得
employees API

利用エンドポイント

GET /employees

レスポンス構造 (GASパース用)

{
  "employees": [
    {
      "id": "uuid-string",
      "number": "EMP001",
      "name": "髙田 勝一",
      "is_active": true
    }
  ]
}

取得した id は、日次勤怠取得時のフィルタリングキーとして利用します。

OPERATIONAL FLOW

04. 取得対象期間の定義

実行タイミングと対象範囲

実行期間データ取得範囲運用目的
毎月 1日 ～ 5日前月 1日 ～ 末日月次確定前の最終チェック
毎月 6日 ～ 末日当月 1日 ～ 前日月中の不備早期修正

GAS側で new Date() を基に from_date と to_date を動的に生成します。

OPERATIONAL FLOW

05. シート構成と役割

1. [RAW] 日次勤怠シート

APIから取得した全データを展開。不備のない正常データも監査ログとして保持します。

2. [LOG] 不備履歴シート

判定ロジックにより「不備」とみなされた行のみを蓄積。Slackへの通知重複を防ぐための「通知済みフラグ」を管理します。

3. [SET] 設定シート

Slack Webhook URLや通知先チャンネル、対象外とする従業員ID等を管理します。
