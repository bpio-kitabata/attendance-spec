Jinjer API 連携
自動勤怠チェックシステム 仕様書

設計の基本方針

Jinjer APIから取得した「打刻実績」と「休暇申請」を突合し、勤怠不備を自動検知します。深夜の自動トリガー実行を基盤とし、管理者の目視確認を不要にする「全自動運用」を目的とした設計です。

Primary Mode
Daily Auto-Trigger
Control Method
Scheduled Sync
SPECIFICATION 01

打刻実績取得仕様 (/v2/attendances)

実績データを「打刻日」単位で取得。ページネーションを完遂させ、全従業員の打刻有無を確定させます。

項目名キー (JSON)役割
打刻日付dateデータ結合の主軸。yyyy-MM-dd。
出勤時刻attended_atyyyy-MM-dd hh:mm:ss。打刻の有無を判定。
退勤時刻left_atyyyy-MM-dd hh:mm:ss。打刻の有無を判定。
欠勤フラグis_absentboolean。未申請欠勤の確認に使用。
SPECIFICATION 02

休暇申請取得仕様 (/v2/employees/requested-day-offs)

休暇申請を取得し、打刻がない日の正当性を検証します。ステータスに基づき、確定実績および申請中データを区別して受理します。

項目名キー (JSON)役割
休暇希望日requested_date打刻データの date と突合。
休暇名holiday_name有給休暇等の具体的な名称。
ステータスstatus1 (承認待ち) または 2 (承認済み) を突合対象として受理。
LOGIC

取得対象期間の定義

実行日による自動切り替え

自動トリガー実行時の日付に応じて、取得範囲を動的に設定。月中は日次是正、月初は前月是正を目的に最適化します。

毎月 1日 ～ 5日 (前月是正フェーズ)

対象: 前月1日 ～ 前月末日

目的: 前月勤怠の締めチェック。 確定に向けたデータの最終整合性確保。

毎月 6日 ～ 末日 (日次是正フェーズ)

対象: 当月1日 ～ 実行日前日

目的: 日次の勤怠是正。 発生した不備を早期に検知し、即時修正を促進。

シート構成と役割

1. 日次勤怠シート

※自動同期のたびに clearContents() し、最新のAPIデータを全件上書き展開。

列カラム名データソース / 加工内容
A従業員名last_name + first_name
B日付yyyy-MM-dd
C出勤時刻attended_at (HH:mm)
D退勤時刻left_at (HH:mm)
E休暇区分holiday_name。ステータスが「1:承認待ち」の場合は名称の末尾に（承認待ち）と追記。
F判定結果不備判定ロジックに基づくステータス

2. 勤怠不備シート

不備（「正常」以外）が発生した履歴を蓄積。通知状況を管理し、同一不備への二重通知を防止する制御用。

操作用シート (手動指定)

特殊な事情により、自動判定以外の特定月を再同期したい場合に使用するオプション画面です。

設定項目機能説明
対象年月指定セルyyyy-MM形式で入力。空欄時は自動期間設定に従う。
実行ボタン指定した月の全データをAPIから強制的に再取得し、シートを最新化。
